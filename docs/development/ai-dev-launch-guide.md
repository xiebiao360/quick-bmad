# AI 可执行开发环境启动与验证指南（模板）

> 这是项目绑定文件模板。安装 BMAD 后，请按你的项目服务/端口/启动方式补齐。
>
> 约定：本文档中的 `${PROJECT_ROOT}` 代表项目根目录。
> 建议执行前设置：
>
> ```bash
> export PROJECT_ROOT="$(git rev-parse --show-toplevel)"
> ```

---

## 验证场景配置

### 场景定义

| 场景 | 指令关键词 | 验证范围 |
|------|-----------|---------|
| 基础启动 | "启动开发环境" | 启动服务 + 基础健康检查 |
| 环境验证 | "验证环境" | 启动服务 + 基础页面/API 验证 |
| 功能验证 | "验证 [功能名]" | 路由到对应深度验证 |

### 功能验证路由（示例）

| 功能名关键词 | 路由到 | 验证内容 |
|-------------|--------|---------|
| member / 会员 | 会员管理验证 | Admin Web 会员列表 |

---

## 前置检查

- Rust / Node / pnpm / Java / Android SDK（按项目需要）

---

## Phase 1: 后端服务启动（模板）

- 代码编译检查
- 端口占用检查
- 启动核心服务（如 Gateway/认证服务）
- 启动业务服务（按场景选择）

健康检查（示例）：
```bash
curl -s http://localhost:<port>/health | head -50
```

---

## Phase 2: Admin Web 启动与验证（模板）

- 安装依赖
- 启动 dev server
- 浏览器访问检查
- 控制台错误检查

---

## 执行报告生成

建议输出结构：

- 总体状态表（各组件 PASS/FAIL/NOT EXECUTED）
- 验证详情（命令/步骤/输出）
- 问题与建议（含根因/风险/下一步）
